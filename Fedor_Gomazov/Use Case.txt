UC-001: Фиксация нарушения безопасности через Telegram-бота
**Связь с User Story:** US-001

**Участники:**
- Основной актор: Специалист по охране труда
- Вторичные акторы: 
  - Telegram-бот
  - Система формирования предписаний 
  - База данных нарушений
  - Уведомительная система

**Предусловия:**
1. Бот активирован и авторизован для специалиста
2. У специалиста есть доступ к месту нарушения
3. Система формирования отчетов доступна

**Ограничения:**
1. Максимальный размер фото/видео - 10MB
2. Геолокация определяется с точностью до 50 метров
3. Время фиксации - не позднее 24 часов с момента нарушения
4. LLM должен поддерживать минимум 5 категорий нарушений
5. Время обработки NLP запроса - не более 5 секунд
6. Точность классификации - не менее 85%

**Триггер:** Специалист отправляет команду /report в Telegram-боте

**Основной сценарий:**
1. Специалист отправляет сообщение с описанием нарушения
2. LLM анализирует текст и определяет тип нарушения
3. Бот запрашивает подтверждение классификации
4. Специалист подтверждает или корректирует тип нарушения
5. Бот запрашивает дополнительные детали (при необходимости)
6. Бот запрашивает фото/видео подтверждение
7. Специалист отправляет медиафайл (опционально)
8. Бот автоматически определяет геолокацию и время
9. Система сохраняет полный отчет о нарушении
10. LLM формирует предписание с рекомендациями
11. Отправляется уведомление ответственному лицу
12. Бот подтверждает успешную фиксацию нарушения

**Альтернативные сценарии:**
- п.2: Если LLM не может классифицировать нарушение:
  1. Бот запрашивает уточняющую информацию
  2. Специалист предоставляет дополнительные детали
  3. LLM повторно анализирует ввод
  4. Если классификация невозможна - записывается как "Неопределенное нарушение"

- п.6: Если медиафайл не предоставлен:
  1. Бот отмечает отчет как "без визуального подтверждения"
  2. Продолжает основной сценарий (п.7)

- п.7: Если геолокация недоступна:
  1. Бот запрашивает ручной ввод местоположения
  2. Специалист указывает место нарушения текстом
  3. Продолжает основной сценарий (п.8)

**Постусловия:**
1. Нарушение зафиксировано в системе
2. Сформировано предписание с рекомендациями
3. Ответственное лицо получило уведомление
4. Отчет доступен для просмотра и скачивания
